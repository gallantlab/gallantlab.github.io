GitHub repository ↗

Our github repository contains lots of open code that may be useful for scientific computing in general, or for fitting encoding models to fMRI or neurophysiology data sets.

Himalaya ↗

Himalaya implements machine learning linear(ized) models in Python, focusing on computational efficiency for large numbers of targets. Himalaya efficiently estimates linear(ized) models on large numbers of targets (for example, thousands of voxels in an fMRI experiment), it runs on both CPU and GPU hardware, and it provides estimators that are compatible with scikit-learn's API. Himalaya is routinely used in our lab to fit voxelwise encoding models to very large fMRI data sets. A paper describing Himalaya can be found here.

Autoflatten ↗

Autoflatten is a Python pipeline for automatically flattening cortical surfaces generated by FreeSurfer. Prior to the development of this pipeline flattening was done largely by hand, an incredibly time-consuming and frustrating process. This new pipeline automates most of the work.

Pycortex ↗

Pycortex is a python-based toolkit for surface visualization of fMRI data. (It can also be used to visualize other types of volumetric brain data.) The brain viewers on this site were all generated using Pycortex. The 2015 publication describing Pycortex can be found here. Documentation for Pycortex can be found here.

It's Complicated ↗

Neuroimaging is moving toward naturalistic, interactive tasks, but their closed-loop nature creates emergent complexity — there's no standard infrastructure for implementation, traditional analysis can't handle correlated data, and extracting design matrices from uncontrolled experiences is inherently difficult. This repository, based on our fMRI driving/navigation experiments, provides a practical guide for designing complex interactive experiments and transforming raw experimental chaos into usable design matrices for tools like banded ridge regression, primarily aimed at game engines but applicable to other complex designs.

CottonCandy ↗

CottonCandy is a scientific library for storing and accessing numpy array data on an S3-compatible cloud storage instance. This is achieved by reading arrays from memory and downloading arrays directly into memory. This means that you don't have to download your array to disk, and then load it from disk into your python session. A paper describing CottonCandy can be found here.

Literature Review Toolkit ↗

The Literature Review Toolkit is scaffolding that drives a Claude (or other LLM) agent through a structured literature review — and keeps it honest. It automates the mechanical work while preserving human judgment for the decisions that matter: every citation is verified against PubMed, PMC, CrossRef, and arXiv to catch hallucinations, references are canonicalized into APA-7 format, and citation counts are reconciled across databases. It can work outward from a research question or start from a lab's publication corpus, and produces a verified annotated bibliography, interactive lineage figures placing theoretical families on a citation-weighted timeline, and an optional AI-authored narrative review.
