{{< content-card title="Himalaya" url="https://github.com/gallantlab/himalaya" image="/img/other/himalaya.webp" alt="himalaya" description="Himalaya implements machine learning linear(ized) models in Python, focusing on computational efficiency for large numbers of targets. Himalaya efficiently estimates linear(ized) models on large numbers of targets (for example, thousands of voxels in an fMRI experiment), it runs on both CPU and GPU hardware, and it provides estimators that are compatible with scikit-learn's API. Himalaya is routinely used in our lab to fit voxelwise encoding models to very large fMRI data sets. A paper describing Himalaya can be found <a href='https://www.sciencedirect.com/science/article/pii/S1053811922008497'>here</a>." >}}

{{< content-card title="Autoflatten" url="https://gallantlab.org/autoflatten/" image="/img/other/autoflatten.webp" alt="autoflatten" description="Autoflatten is a Python pipeline for automatically flattening cortical surfaces generated by FreeSurfer. Prior to the development of this pipeline flattening was done largely by hand, an incredibly time-consuming and frustrating process. This new pipeline automates most of the work." >}}

{{< content-card title="Pycortex" url="https://github.com/gallantlab/pycortex" image="/img/other/pycortex.webp" alt="pycortex" description="Pycortex is a python-based toolkit for surface visualization of fMRI data. (It can also be used to visualize other types of volumetric brain data.) The brain viewers on this site were all generated using Pycortex. The 2015 publication describing Pycortex can be found <a href='https://www.frontiersin.org/articles/10.3389/fninf.2015.00023/full'>here</a>. Documentation for Pycortex can be found <a href='https://gallantlab.github.io/pycortex/'>here</a>." >}}

{{< content-card title="It's Complicated" url="https://gallantlab.org/Its-Complicated/" image="/img/other/itscomplicated.logo.webp" alt="It's Complicated logo" description="Neuroimaging is moving toward naturalistic, interactive tasks, but their closed-loop nature creates emergent complexity — there's no standard infrastructure for implementation, traditional analysis can't handle correlated data, and extracting design matrices from uncontrolled experiences is inherently difficult. This repository, based on our fMRI driving/navigation experiments, provides a practical guide for designing complex interactive experiments and transforming raw experimental chaos into usable design matrices for tools like banded ridge regression, primarily aimed at game engines but applicable to other complex designs." >}}

{{< content-card title="CottonCandy" url="https://github.com/gallantlab/cottoncandy" image="/img/other/cottoncandy.webp" alt="cottoncandy" description="CottonCandy is a scientific library for storing and accessing numpy array data on an S3-compatible cloud storage instance. This is achieved by reading arrays from memory and downloading arrays directly into memory. This means that you don't have to download your array to disk, and then load it from disk into your python session. A paper describing CottonCandy can be found <a href='https://joss.theoj.org/papers/10.21105/joss.00890.pdf'>here</a>." >}}

{{< content-card title="Literature Review Toolkit" url="https://gallantlab.org/literature-review-toolkit/" image="/img/other/literature-review-toolkit.webp" alt="Literature Review Toolkit lineage figure" description="The Literature Review Toolkit is scaffolding that drives a Claude (or other LLM) agent through a structured literature review — and keeps it honest. It automates the mechanical work while preserving human judgment for the decisions that matter: every citation is verified against PubMed, PMC, CrossRef, and arXiv to catch hallucinations, references are canonicalized into APA-7 format, and citation counts are reconciled across databases. It can work outward from a research question or start from a lab's publication corpus, and produces a verified annotated bibliography, interactive lineage figures placing theoretical families on a citation-weighted timeline, and an optional AI-authored narrative review." >}}

{{< content-card title="SharpEyes" url="https://gallantlab.org/SharpEyes/" image="/img/other/sharpeyes.webp" alt="SharpEyes Stimulus & Gaze window, showing gaze overlaid on a stimulus video" description="SharpEyes is a desktop application for eyetracking and for building motion-energy features out of stimulus videos. It finds the pupil in every frame of an eyetracking video using semi-automatic template matching, and every result is hand-editable so you can ensure data quality. Gaze can then be overlaid on the stimulus to check and correct the eyetraces, and the stimulus video can be recentered on gaze to produce a retinotopic video approximating what visual cortex actually sees. Finally, SharpEyes computes motion-energy features from the stimulus and lets you visualize the filters in stimulus space; the features are computed by <a href='https://github.com/gallantlab/pymoten'>PyMoten</a> under the hood, but you never have to touch Python. Precompiled Windows and Linux builds are on the <a href='https://github.com/gallantlab/SharpEyes/releases'>releases page</a>, and the source is on <a href='https://github.com/gallantlab/SharpEyes'>GitHub</a>." >}}
